Lecture Notes for CS61B

Lecture 1

Object-oriented programming

  • Object: An object is a repository of data.
  • Class : A class is a type of object.
  • Method : A procedure or function that operates on an object or a class.
  • Inheritance : A class may inherit properties from a more general class.
  • Polymorphism : The ability to have one method call work on several different classes of objects.

The one thing that truly distinguishes object-oriented languages from procedural languages (C, Fortran, Basic, Pascal) is polymorphism.

Lecture 8

The “public” and “private” Keywords

A private method or field is invisible and inacessible to other classes, and can be used only within the class in which the field or method is declared.

The interface of a class is a set of prototypes for public methods plus descriptions of the methods’ behaviors.

An Abstract Data Type is a class that has a well-defined interface, but its implementation details are firmly hidden from other classes.

An invariant is a fact about a data structure that is always true, no matter what methods are called by external classes.

Lecture 9

Stack and Heap

The heap stores all objects, including all arrays, and all class variables.

The stack stores all local variables, including all parameters.

When a method is called, the JVM creates a stack frame that stores the parameters and local variables for that method.

The java.lang library has a method “Thread.dumpStack“ the prints a list of the methods on the stack.

Parameter Passing

Java passes all parameters by value. This means that the method has copies of the actual parameters, and cannot change the originals. The copies reside in the method’s stack frame for the method. The method can change these copies, but the original values that were copied are not changed.

Lecture 10

Inheritance

A subclass can modify or augment a superclass in at least three ways:

  1. It can declare new fields
  2. It can declare new methods
  3. It can override old methods with new implementations

Inheritance and Constructors

The zero-parameter SList() constructor is always called by default, regardless of the parameters passed to the TailList constructor.

To change this default behavior, the TailList constructor can explicitly call any constructor for its superclass by using the “super” keyword.

Invoking Overridden Methods

Unlike superclass constructor, ordinary superclass method invocations need not be the first statement in a method.

The “protect” Keyword

“protected” is a level of protection somewhere between “public” and “private”. A “protected” field is visible to the declaring class and all its subclasses, but not to other class. “private” fields aren’t even visible to the subclasses.

If “head” and “size” are declared private, the method TailList.insertFront can’t access them and won’t compile.

Equals()

inherit from Object.equals(), to compare two reference

Four different degrees of equality:

  1. Reference equality, ==
  2. Shallow structural equality. Two objects are “equals” if all their fields are ==
  3. Deep structural eauality. Two objects are “equals” if all their fields are “equals”. For example, two SLists that represent the same sequence of items (though the SListNodes may be different). 也就是说两个Linked-List一样, 只需要这两个Linked-List每个Node的值一样, 不需要完全一样的Node对应一样的值.
  4. Logical equality.
    • Two “Set” objects are “equals” if they contain the same elements, even if the underlying lists store the elements in different orders.
    • The Fractions 1/3 and 2/6 are “equals”, even though their numerators

Overriding doesn’t work if we change the signature of the original method, even just to change a parameter to a subclass

Lecture 12

Abstract Class

an abstract class is a class whose sole purpose is to be extended.

A non-abstract class may never contain an abstract method, nor inherit one without providing an implementation.

An abstract class lets you define an interface 1) for multiple classes to share. 2) without defining any of them yet.

public void ListSorter(List l){...}

Java Interfaces

A java interface could not implement any methods, nor can it include any fields expect “final static” constants. It only contains method, prototypes and constants.

The method prototypes in a Java interface may be declared without the “abstract” keyword.

Interfaces can be extended with subinterfaces. A subinterface can have multiple superinterfaces, so we can group several interfaces into one.

public interface NukeAndCompare extends Nukeable, Comparable{}
